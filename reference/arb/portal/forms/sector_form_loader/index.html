arb.portal.forms.sector_form_loader

Form registry and dynamic loader for sector-specific feedback forms.

This module defines a centralized registry (FORM_REGISTRY) that maps sector names (e.g., "landfill", "oil_and_gas") to their corresponding WTForms classes. It enables dynamic form loading in views or helper functions by passing a sector identifier string.

Benefits:

  • Avoids long if-elif chains when selecting forms by sector.
  • Centralizes form declarations for maintainability.
  • Supports dynamic sector resolution (e.g., from user input or URL parameters).
  • Scales cleanly to additional sectors (e.g., "dairy", "energy").

Example:

from arb.portal.forms.sector_form_loader import get_sector_form_class

sector_name = "landfill"
FormClass = get_sector_form_class(sector_name)
form = FormClass() if FormClass else None

Notes:

  • All referenced forms must be explicitly imported and registered here.
  • If an unknown sector is passed to get_sector_form_class, the result will be None.

def get_sector_form_class(sector_name: str):
  """
  Retrieve the WTForms class associated with a given sector name.

  Args:
    sector_name (str): Logical string identifier for the sector (e.g., "landfill").

  Returns:
    type[FlaskForm] | None: Corresponding WTForms subclass if found; otherwise None.
  """
  return FORM_REGISTRY.get(sector_name)
